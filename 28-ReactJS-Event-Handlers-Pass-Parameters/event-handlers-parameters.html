Lesson Twenty-Eight: React Event Handlers Pass Parameters

الدرس الثامن والعشرون: تمرير بارامترات مع Event Handlers في React

مقدمة عن تمرير البارامترات

في هذا الدرس سنتعلم كيفية تمرير البارامترات (Parameters) إلى الدوال (Methods) في معالجات الأحداث (Event Handlers) في React. هذه المهارة مهمة جداً عندما نحتاج إلى تخصيص سلوك المكونات (Components) بناءً على قيم مختلفة.

طرق تمرير البارامترات

هناك طريقتان أساسيتان لتمرير البارامترات إلى معالجات الأحداث في React:

  1. استخدام Arrow Function
  2. استخدام طريقة Bind

استخدام Arrow Function

الطريقة الأكثر شيوعاً وسهولة هي استخدام Arrow Function:


// تعريف الدالة
handleClick = (message) => {
    alert(message);
}

// استخدام الدالة في الزر
<button onClick={() => this.handleClick("مرحباً")}>
    اضغط هنا
</button>

استخدام Bind

يمكن أيضاً استخدام طريقة bind:


<button onClick={this.handleClick.bind(this, "مرحباً")}>
    اضغط هنا
</button>

الأخطاء الشائعة

من الأخطاء الشائعة استدعاء الدالة مباشرة مع القوسين:


// ❌ خطأ - سيتم تنفيذ الدالة مباشرة
<button onClick={this.handleClick("مرحباً")}>

// ✅ صحيح - سيتم تنفيذ الدالة عند النقر
<button onClick={() => this.handleClick("مرحباً")}>

أمثلة عملية

مثال عملي لقائمة منتجات:


class ProductList extends React.Component {
    // دالة معالجة حدث الشراء
    handleBuy = (productId, price) => {
        console.log(`شراء المنتج ${productId} بسعر ${price}`);
    }

    render() {
        return (
            <div>
                // أزرار المنتجات مع تمرير معرف المنتج والسعر
                <button onClick={() => this.handleBuy(1, 99.99)}>
                    شراء المنتج 1
                </button>
                <button onClick={() => this.handleBuy(2, 149.99)}>
                    شراء المنتج 2
                </button>
            </div>
        );
    }
}

ملخص الدرس

  • استخدم Arrow Function لتمرير البارامترات بشكل مرن ومقروء
  • تجنب استدعاء الدالة مباشرة مع القوسين في الـ onClick
  • يمكنك تمرير أكثر من بارامتر للدالة حسب الحاجة
