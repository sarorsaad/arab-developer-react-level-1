بناء موقع بدون Components

بسم الله الرحمن الرحيم

في هذا الدرس سنقوم ببناء موقع بسيط بدون استخدام Components، ثم في الدرس القادم سنقوم بإعادة كتابته باستخدام Components لنرى الفرق بين الطريقتين.

In this lesson, we'll build a simple website without using Components, then in the next lesson, we'll refactor it using Components to see the difference between both approaches.

إعداد المشروع | Project Setup

نبدأ بإنشاء مشروع React جديد باستخدام Vite:

We start by creating a new React project using Vite:

# إنشاء مشروع جديد
npm create vite@latest my-app

# الدخول إلى مجلد المشروع
cd my-app

# تثبيت المكتبات
npm install

# تشغيل المشروع
npm run dev

هيكل الموقع | Website Structure

موقعنا يتكون من 4 أجزاء رئيسية:

Our website consists of 4 main parts:

  • Header (الهيدر)
    الجزء العلوي من الموقع
    The top section of the website
  • Main Content (المحتوى الرئيسي)
    منطقة المحتوى الرئيسية
    The main content area
  • Sidebar (الشريط الجانبي)
    قائمة جانبية للتنقل
    Navigation sidebar
  • Footer (الفوتر)
    الجزء السفلي من الموقع
    The bottom section of the website

تنفيذ الكود | Code Implementation

سنقوم بكتابة كل الأجزاء في ملف main.jsx:

We'll write all parts in main.jsx file:

import React from 'react';
import { createRoot } from 'react-dom/client';

// إنشاء جذر التطبيق | Create root element
createRoot(document.getElementById('root')).render(
  <div className="parent">
    {/* Header */}
    <header>
      <h1>Welcome to my website</h1>
    </header>

    {/* Main Content */}
    <main className="main-content">
      <h2>Main Content</h2>
      <p>This is the main content area of our website.</p>
    </main>

    {/* Sidebar */}
    <aside className="sidebar">
      <h3>Navigation</h3>
      <ul>
        <li><a href="#">Link 1</a></li>
        <li><a href="#">Link 2</a></li>
        <li><a href="#">Link 3</a></li>
      </ul>
    </aside>

    {/* Footer */}
    <footer>
      <p> 2024 My Website. All rights reserved.</p>
    </footer>
  </div>
);

ملاحظة: لاحظ أننا نكتب كل الكود في ملف واحد، مما يجعل الكود أقل تنظيماً وأصعب في الصيانة.

Note: Notice that we're writing all code in one file, which makes the code less organized and harder to maintain.
