What Are Default Props in React?

شرح Default Props في React بالتفصيل

بسم الله الرحمن الرحيم

في هذا الدرس سنتعلم كيف نتعامل مع الحالات التي لا يتم فيها تمرير قيم للـ Props، وكيف نضع قيم افتراضية لها باستخدام Default Props.

In this lesson, we'll learn how to handle cases where Props values are not passed, and how to set default values using Default Props.

مشكلة القيم الافتراضية | The Problem with Missing Props

عندما لا نقوم بتمرير قيمة لـ Prop معين، قد نواجه مشاكل مثل:

  • ظهور قيم فارغة في واجهة المستخدم
  • أخطاء في العمليات الحسابية
  • سلوك غير متوقع في التطبيق
// مثال لمكون بدون قيم افتراضية
// Component without default values
<Project
title="Admin Dashboard"
description="A powerful dashboard"
// numberOfDays is missing!
/>

مفهوم Default Props | Understanding Default Props

Default Props هي آلية في React تتيح لنا تحديد قيم افتراضية للـ Props في حالة عدم تمريرها للمكون.

Default Props is a React mechanism that allows us to specify default values for Props when they are not passed to the component.

// تعريف القيم الافتراضية باستخدام static defaultProps
// Defining default values using static defaultProps
class Project extends React.Component {
static defaultProps = {
numberOfDays: 0,
technologies: []
};
// ... rest of the component
}

Static Default Props | الخصائص الثابتة للقيم الافتراضية

كلمة static تعني أن هذه القيم خاصة بالـ Class نفسه وليست بالـ Instance:

  • القيم الافتراضية ثابتة لجميع نسخ المكون
  • يتم استخدامها فقط عند عدم تمرير قيمة للـ Prop
  • تساعد في تجنب الأخطاء وتحسين تجربة المستخدم

مثال عملي | Practical Example

// مكون Project مع القيم الافتراضية
// Project component with default values
class Project extends React.Component {
static defaultProps = {
numberOfDays: 0,
technologies: [],
description: 'No description available'
};
render() {
return (
<div className="project">
<h3>{this.props.title}</h3>
<p>{this.props.description}</p>
<p>Days worked: {this.props.numberOfDays}</p>
</div>
);
}
}
