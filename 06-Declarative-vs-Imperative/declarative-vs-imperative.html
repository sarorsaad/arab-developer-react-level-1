Lesson Six: Declarative vs Imperative Programming

الدرس السادس: البرمجة التصريحية مقابل الإجرائية

البرمجة التصريحية مقابل الأمرية (Declarative vs Imperative Programming)

هناك نمطان رئيسيان في البرمجة (Two main programming paradigms):

  • البرمجة الأمرية (Imperative): تخبر الحاسوب كيف يفعل شيئاً (Tell computer HOW to do something)
  • البرمجة التصريحية (Declarative): تخبر الحاسوب ماذا تريد (Tell computer WHAT you want)

البرمجة الأمرية (Imperative Programming)

في البرمجة الأمرية (In Imperative Programming):

  • تحدد كل خطوة بالتفصيل (Specify each step in detail)
  • تتحكم في تدفق البرنامج (Control program flow)
  • تدير الحالة يدوياً (Manually manage state)

// البرمجة الإجرائية (Imperative)
const numbers = [1, 2, 3, 4, 5];
const doubledNumbers = [];

for (let i = 0; i < numbers.length; i++) {
    doubledNumbers.push(numbers[i] * 2);
}

console.log(doubledNumbers); // [2, 4, 6, 8, 10]

البرمجة التصريحية (Declarative Programming)

في البرمجة التصريحية (In Declarative Programming):

  • تصف ما تريد (Describe what you want)
  • تدع المنصة تهتم بالتفاصيل (Let platform handle details)
  • تركز على النتيجة النهائية (Focus on end result)

// البرمجة التصريحية (Declarative)
const numbers = [1, 2, 3, 4, 5];
const doubledNumbers = numbers.map(num => num * 2);

console.log(doubledNumbers); // [2, 4, 6, 8, 10]

مقارنة بين النمطين (Comparison)

البرمجة الأمرية (Imperative)

  • تحدد الخطوات (Steps)
  • تدير التفاصيل (Details)
  • تتحكم في التدفق (Flow)
  • مثل: Pure JavaScript

البرمجة التصريحية (Declarative)

  • تحدد النتيجة (Result)
  • تترك التفاصيل للمنصة (Platform handles details)
  • تركز على الوصف (Description)
  • مثل: React, SQL

مثال في React (React Example)

مثال عملي يوضح الفرق (Practical example showing the difference):


// مثال على البرمجة الإجرائية في React
function ImperativeCounter() {
    let count = 0;
    
    function handleClick() {
        count++;
        document.getElementById('counter').textContent = count;
    }
    
    return (
        <div>
            <h1 id="counter">0</h1>
            <button onClick={handleClick}>
                زيادة
            </button>
        </div>
    );
}

// مثال على البرمجة التصريحية في React
function DeclarativeCounter() {
    const [count, setCount] = React.useState(0);
    
    return (
        <div>
            <h1>{count}</h1>
            <button onClick={() => setCount(count + 1)}>
                زيادة
            </button>
        </div>
    );
}

فوائد البرمجة التصريحية (Benefits of Declarative Programming)

  • كود أنظف وأسهل للقراءة (Cleaner, more readable code)
  • أقل عرضة للأخطاء (Less prone to errors)
  • أسهل للصيانة (Easier to maintain)
  • أسهل للاختبار (Easier to test)
  • إعادة استخدام أفضل للكود (Better code reusability)
