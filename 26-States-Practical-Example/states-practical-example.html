Lesson Twenty-Six: States Practical Example

الدرس السادس والعشرون: مثال عملي على States

بسم الله الرحمن الرحيم

في هذا الدرس، سنتعلم كيفية تحسين كودنا باستخدام States من خلال مثال عملي. سنرى كيف يمكن للـ States أن تجعل كودنا أكثر مرونة وقابلية للصيانة.

In this lesson, we'll learn how to improve our code using States through a practical example. We'll see how States can make our code more flexible and maintainable.

الكود الأولي | Initial Code

لنبدأ بمثال بسيط يحتاج إلى تحسين:


class AgeChecker extends React.Component {
    render() {
        return (
            <div>
                <p>Your age is: {this.props.age}</p>
                <p>Your age after increment is: {this.props.age + 3}</p>
                <p>
                    {this.props.age + 3 > 30 ? 'Not Accepted' : 'Accepted'}
                </p>
            </div>
        );
    }
}

المشكلة في الكود | The Problem

المشكلة في الكود السابق:

  • تكرار العملية الحسابية (this.props.age + 3)
  • صعوبة تغيير قيمة الزيادة (3) في أكثر من مكان
  • عدم مرونة الكود عند الحاجة للتعديل

إعادة كتابة الكود باستخدام States | Refactoring with States

لنحسن الكود باستخدام States:


class AgeChecker extends React.Component {
    state = {
        ageAfterIncrement: this.props.age + 3
    }

    render() {
        return (
            <div>
                <p>Your age is: {this.props.age}</p>
                <p>Your age after increment is: {this.state.ageAfterIncrement}</p>
                <p>
                    {this.state.ageAfterIncrement > 30 ? 'Not Accepted' : 'Accepted'}
                </p>
            </div>
        );
    }
}

العرض الشرطي في React | Conditional Rendering in React

في React، نستخدم العامل الثلاثي (Ternary Operator) للعرض الشرطي:

// الصيغة العامة
condition ? valueIfTrue : valueIfFalse

// مثال
{this.state.ageAfterIncrement > 30 ? 'Not Accepted' : 'Accepted'}

فوائد استخدام States | Benefits of Using States

  • تخزين القيم المحسوبة مرة واحدة | Store calculated values once
  • سهولة التعديل على الكود | Easy code modification
  • تقليل التكرار | Reduce redundancy
  • تحسين قابلية الصيانة | Improve maintainability

الملخص | Summary

تعلمنا في هذا الدرس كيفية تحسين كودنا باستخدام States. رأينا كيف يمكن للـ States أن تساعدنا في تقليل التكرار وجعل كودنا أكثر مرونة وقابلية للصيانة.

In this lesson, we learned how to improve our code using States. We saw how States can help us reduce redundancy and make our code more flexible and maintainable.
